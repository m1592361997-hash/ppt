# 第二轮视觉精修规范

## 技术边界

- 技术栈保持 Vite、React、TypeScript、轻量 Hash 路由与原生 CSS。
- 不增加远程字体、图片、外部 API、动画库或 UI 组件库。
- `src/data/portfolio.ts` 继续作为唯一内容数据源，本轮不修改其中的文案、项目顺序与图片顺序。

## 视觉方向

- 70% 编辑式极简：瑞士网格、粗体中文标题、非对称留白、细线索引。
- 20% 弥散渐变：多层径向渐变、低饱和模糊色块、细噪点。
- 10% 轻量动效：标题逐行进入、滚动显现、图片轻微缩放与箭头位移。

## 实施原则

- 首页 Hero 保持原内容，以更强的中文标题比例和跨网格产品界面建立第一视觉。
- 四个项目保留统一信息结构，但通过列宽、图文顺序、背景与留白形成不同构图。
- 标签从大量胶囊改为编辑式斜线分隔；状态、占位与操作控件仍保留清晰边界。
- 案例首屏使用 12 列重叠构图，章节用稳定的左侧索引和更有层次的纵向节奏。
- 动效仅使用 `transform` 与 `opacity`，并完整支持 `prefers-reduced-motion`。
- 重点检查 1440、1280、1024、768 与 390 像素宽度，整页不得横向溢出。
